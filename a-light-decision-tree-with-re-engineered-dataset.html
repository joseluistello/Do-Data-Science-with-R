1.5 A Light Decision Tree with Re-engineered Dataset

Let us see if we can improve on this with a consideration of the correlation analysis and PCA analyses results. We only use the three most related attributes: Sex, Pclass, Fare_pp.

model4 <- rpart(Survived ~ Sex + Fare_pp + Pclass,
              data=RE_train,
              method="class")

#assess prediction accuracy on train data
Predict_model4_train <- predict(model4, RE_train, type = "class")
conMat <- confusionMatrix(as.factor(Predict_model4_train), as.factor(RE_train$Survived))
conMat$table

Our assessment on model4’s accuracy on the train data shows model4’s accuracy is 81.48%. It actually reduced the prediction accuracy quite a bit from the model3, which was 85.97%. Let us use this model to make another prediction on the test dataset and see if the accuracy on the test dataset is also decreased a lot.

Prediction4 <- predict(model4, RE_test, type = "class")
submit4<- data.frame(PassengerId = RE_test$PassengerId, Survived = Prediction4)
write.csv(submit4, file = "tree_model4.CSV", row.names = FALSE)

After submit to Kaggle the feedback was 0.75837. Despite the accuracy on train dataset has been decreased almost 4.5%, There is a little improvement on the test dataset. Let us look into the difference between the last two predictions,
